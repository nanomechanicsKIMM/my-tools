# 특허 조사·분석 실행 가이드 (일반)

**적용 범위**: 다양한 연구·기술 분야의 RFP에 공통 적용. 특정 기술에 한정하지 않음.  
**CSV**: 사용자가 Google Patents에서 검색 후 **직접 다운로드**하여 제공. (Playwright 자동 다운로드는 보류.)

---

## 1. 검색식·URL 생성

RFP 마크다운 경로와 (선택) 기술 대분류를 지정하여 검색식과 URL을 만든다.

```bash
cd .codex\skills\patent-strategy-report\scripts
uv run python generate_query.py "<RFP.md 경로>" "[기술분류]" [--years 10]
```

- `--years 10`: 최근 10년 (기본). **사용자가 `--years 12`, `--years 15` 등으로 지정하면 그 값이 우선** 적용됨. 보고서는 CSV에 포함된 전체 기간을 분석한다.
- 출력된 **SEARCH_URL**을 복사해 둔다.

---

## 2. CSV 준비 (사용자)

1. 브라우저에서 위 **SEARCH_URL**을 연다. (Google Patents 검색 결과 페이지가 뜬다.)
2. 결과 목록 **위쪽**의 **Download (CSV)** 버튼을 클릭해 CSV를 저장한다.
3. 저장한 CSV 파일 경로를 확보한다 (예: `output/patent_results.csv`).  
   필요 시 프로젝트의 `output/` 폴더에 넣어 두면 편하다.

---

## 3. 노이즈 필터 (제거율 ≤10% 목표) — 반복 루프의 핵심

- **한 라운드**: `filter_noise.py`가 RFP 키워드로 점수 매겨 노이즈를 제거하고 **REMOVAL_RATIO**를 출력한다.
- **REMOVAL_RATIO > 10%**이면: 스크립트가 **제거된(노이즈) 행**에서 자주 나오는 단어를 분석해 **NOT 후보**를 제안하고, **IMPROVED_QUERY_SUGGESTION**(현재 검색식 + `NOT (term1 OR term2 OR ...)`)을 출력한다. 사용자는 이 **개선된 검색식/URL**로 Google Patents에서 다시 검색·CSV 다운로드 후, 같은 파이프라인을 다시 실행한다. **REMOVAL_RATIO ≤ 10%**가 될 때까지 반복한다.

**필터만 실행할 때** (검색식 전달 시 개선안 출력):

```bash
uv run python filter_noise.py "<CSV경로>" "<RFP.md 경로>" -o output/filtered.csv [--threshold 0.15] [--current-query "<이번에 사용한 검색식>"]
```

**한 번에 실행할 때** (`run_full_pipeline.py`): `--search-query`와 `--search-url`을 넘기면, REMOVAL_RATIO > 10%인 경우 **개선된 검색식·URL**을 출력하고 **집계/보고서 단계 전에 중단**한다. 사용자가 새 CSV를 받아 같은 명령으로 다시 실행하면 된다.

---

## 4. 집계 및 보고서 데이터 생성

```bash
uv run python aggregate_csv_report.py output/filtered.csv -o output --report-title "<보고서 제목>" --search-query "<검색식>" --search-url "<SEARCH_URL>" [--rfp-path "<RFP.md>"]
```

- `--report-title`: RFP 주제에 맞는 제목 (분야 무관).
- `--search-query`, `--search-url`: 1번에서 얻은 값.

---

## 5. 보고서 MD 생성

```bash
uv run python fill_report.py output/aggregate_report_data.json -o output/<보고서파일명>.md --topic <주제슬러그>
```

- `<주제슬러그>`: RFP 제목에서 추린 짧은 식별자 (예: `센서융합디스플레이`, `배터리양극재` 등, 분야별로 자유롭게 지정).

---

## 6. 서술문 채우기

생성된 보고서에서 `[ ... ]` 로 된 플레이스홀더를 LLM(Claude 등)으로 채워 최종본을 완성한다.

---

## 한 번에 실행 (CSV 경로만 있을 때)

CSV를 이미 받아 두었다면, 필터 → 집계 → 보고서까지 한 번에 실행할 수 있다.

```bash
cd .codex\skills\patent-strategy-report\scripts
uv run python run_full_pipeline.py "<CSV경로>" [--rfp-path "<RFP.md>"] [--output-dir output] [--report-title "제목"] [--topic 주제] [--search-query "검색식"] [--search-url "URL"]
```

- **--rfp-path**: 노이즈 필터 사용 시 RFP 경로 (없으면 필터 생략).
- **--report-title**, **--topic**: 어떤 연구 분야든 자유롭게 지정.
- **--search-query**, **--search-url**: 1번에서 얻은 값 (보고서 참고 데이터용).
- 특정 사례(예: 15년·특정 RFP)용으로 `run_15yr_pipeline.py` 가 있으면, 해당 RFP·기간에 맞춰 동일 인자를 넘겨 사용할 수 있다.
