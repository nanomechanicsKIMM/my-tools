# 상관성 점수 파이프라인 (실행 가이드)

기존 노이즈 필터 루프를 제거하고, **상관성 점수(포함/제외 가중치)** 기반 알고리즘으로 통일한 워크플로우이다.

---

## 1. 알고리즘 요약

| 단계 | 내용 |
|------|------|
| 1 | 특허 CSV에서 **제목** 기준 상관성 점수 산출. RFP 부합성(TF-IDF 코사인) + **반드시 포함 단어**는 높은 양의 점수, **반드시 제외 단어**는 높은 음의 점수로 계량화. |
| 2 | 상관성 점수(제목 기준) **상위 1만 건**을 별도 파일로 저장 → **특허 전략 보고서의 통계적 부분** 작성에 사용. |
| 3 | 상위 **100건** 추출 → **특허 초록·대표청구항** 수집 (`fetch_abstracts.py`: `abstract`, `representative_claim` 컬럼 추가). |
| 4 | **초록+대표청구항** 기준으로 상관성 점수 산출 (초록 50% + 대표청구항 50%). 동일 포함/제외 가중치 적용 → **상위 10건**을 핵심 특허로 저장. |
| 5 | 핵심 특허 분석 진행 후 특허 전략 보고서에 추가. |

---

## 2. 일괄 실행 (권장)

```bash
cd scripts
python run_relevance_pipeline.py <v1_csv> <rfp_path> -o ../output [--topic 센서융합디스플레이]
```

**옵션**

- `--include-terms "A,B"`: 반드시 포함될 단어 (쉼표 구분). 제목/초록에 있으면 **양의 가중치** 부여.
- `--exclude-terms "C,D"`: 반드시 제외될 단어. 제목/초록에 있으면 **음의 가중치** 부여.
- `--include-weight 0.2`: 포함 단어당 가산 점수 (기본 0.2).
- `--exclude-weight 0.5`: 제외 단어당 감점 (기본 0.5).
- `--topic`: 보고서 제목용 토픽 (예: 센서융합디스플레이).
- `--skip-fetch`: 초록 크롤링 생략 (이미 abstract 컬럼이 있을 때).
- `--fetch-delay 1.0`: 초록 요청 간 대기 시간(초).

**출력 파일 (output 디렉터리)**

| 파일 | 설명 |
|------|------|
| `v1_top10000.csv` | 제목 기준 상관성 상위 10,000건 (통계·보고서용). |
| `top100_for_abstracts.csv` | 상위 100건 (초록 수집 입력). |
| `top100_with_abstracts.csv` | 초록·대표청구항 컬럼이 추가된 100건. |
| `top100_abstract_scored.csv` | 초록+대표청구항(5:5) 기준 점수가 부여된 100건. |
| `핵심특허_상위10건_목록.csv` | 초록 기준 상위 10건 핵심 특허. |
| `aggregate_report_data_10k.json` | 10k 기준 집계 데이터. |
| `{topic}_세계특허현황_분석보고서.md` | 채워진 보고서. |

## 3. 단계별 실행 (수동)

필요 시 아래 순서로 개별 스크립트를 실행할 수 있다.

### 3.1 제목 기준 상관성 점수 → 상위 10k

```bash
python score_title_relevance.py <v1_csv> <rfp_path> -o output/v1_top10000.csv --top 10000
python score_title_relevance.py <v1_csv> <rfp_path> -o output/v1_top10000.csv --top 10000 --include-terms "sensor,display" --exclude-terms "OLED,LCD"
```

### 3.2 상위 100건 추출 후 초록·대표청구항 수집

상위 10k CSV에서 상위 100행을 복사한 CSV를 준비한 뒤:

```bash
python fetch_abstracts.py output/top100_for_abstracts.csv -o output/top100_with_abstracts.csv --delay 1.0
```

출력 CSV에는 `abstract`와 `representative_claim`(대표청구항, Claim 1) 컬럼이 추가된다.

### 3.3 초록+대표청구항 기준 상관성 점수 (5:5) → 상위 10

```bash
python score_abstract_relevance.py output/top100_with_abstracts.csv <rfp_path> -o output/top100_abstract_scored.csv --top 100
```

이후 `top100_abstract_scored.csv`에서 `relevance_score` 기준 상위 10행을 `핵심특허_상위10건_목록.csv`로 저장.  
점수는 **초록–RFP 코사인 유사도 50% + 대표청구항–RFP 코사인 유사도 50%** 에 포함/제외 가중치를 더한 값이다.

### 3.4 집계 및 보고서

`aggregate_csv_report.py`로 10k CSV 집계 → JSON 생성 → `fill_report.py`로 보고서 MD 생성. (`run_relevance_pipeline.py`가 이 단계를 포함함.)

---

## 4. 관련 스크립트

| 스크립트 | 역할 |
|----------|------|
| `score_relevance_weighted.py` | 공통: TF-IDF 코사인 + 포함/제외 가중치 점수 계산. |
| `score_title_relevance.py` | 제목 기준 상관성 점수, 상위 N건 저장. |
| `score_abstract_relevance.py` | 초록+대표청구항 기준 상관성 점수 (5:5). 포함/제외 가중치 동일 적용. |
| `fetch_abstracts.py` | result link로 초록·대표청구항(Claim 1) 크롤링, `abstract`·`representative_claim` 컬럼 추가. |
| `run_relevance_pipeline.py` | 위 단계 일괄 실행 + 집계·보고서. |

---

*기존 노이즈 필터(`filter_noise.py`, REMOVAL_RATIO 10% 루프)는 사용하지 않는다.*
