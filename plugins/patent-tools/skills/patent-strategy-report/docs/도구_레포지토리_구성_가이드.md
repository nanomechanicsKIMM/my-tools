# 스킬·플러그인을 한 레포지토리에 모아 여러 PC에 구성하기

**목적**: GitHub에 "도구 통합 레포지토리" 하나를 두고, 새 PC에서는 **clone → setup 실행**만으로 Codex 스킬·Cursor 규칙 등 동일한 도구 환경을 만든다.

---

## 1. 레포지토리 구조 제안

한 저장소에 스킬·플러그인·설정을 **하위 폴더**로 모아 둡니다.

```
my-tools/                          # 저장소 루트 (이름은 자유)
├── README.md                      # 이 레포 사용법·설치 절차
├── setup.ps1                      # Windows: 도구 배포 스크립트
├── setup.sh                       # Mac/Linux: 도구 배포 스크립트
├── skills/                        # Codex 스킬들
│   ├── patent-strategy-report/   # 특허 전략 보고서 스킬
│   │   ├── SKILL.md
│   │   ├── scripts/
│   │   ├── templates/
│   │   └── ...
│   └── (다른 스킬 폴더)
├── cursor/                        # (선택) Cursor 관련
│   └── rules/                     # .cursor/rules 에 복사할 규칙들
│       └── *.mdc
└── docs/                          # (선택) 추가 설명
    └── 도구_레포지토리_구성_가이드.md
```

- **skills/**  
  Codex가 읽는 스킬 폴더(`$CODEX_HOME/skills`)에 **그대로 복사할** 내용. 각 하위 폴더 하나가 스킬 하나(예: `patent-strategy-report`).
- **cursor/**  
  Cursor 규칙·플러그인 설정 등을 넣을 때 사용. `setup` 스크립트에서 `%USERPROFILE%\.cursor\rules` 등으로 복사하도록 할 수 있음.
- **setup.ps1 / setup.sh**  
  clone한 뒤 한 번 실행하면, `skills/` 내용을 실제 스킬 경로로 복사(또는 심볼릭 링크)해 주는 스크립트.

---

## 2. 레포지토리 만드는 절차

### 2.1 로컬에 폴더 구조 만들기

```powershell
mkdir my-tools
cd my-tools
mkdir skills
mkdir cursor
# (선택) mkdir cursor\rules, mkdir docs
```

### 2.2 스킬 넣기

기존 스킬 폴더를 **그대로** `skills/` 아래에 복사합니다. `.venv`, 대용량 CSV 등은 제외합니다.

```powershell
# 예: patent-strategy-report 스킬을 skills 아래로
xcopy /E /I "c:\Users\JHKIM\Patent_Analysis\.codex\skills\patent-strategy-report" "my-tools\skills\patent-strategy-report"
# skills\patent-strategy-report\scripts\.venv 는 삭제하거나 .gitignore로 제외
```

- `skills/patent-strategy-report/.gitignore`에 `.venv/`, `output/*.csv` 등이 있으면 그대로 두어도 됨.
- 다른 스킬이 있으면 같은 방식으로 `skills/other-skill-name/` 형태로 추가.

### 2.3 배포 스크립트 작성

**setup.ps1** (Windows): clone한 레포 루트에서 실행 시 `skills/`를 Codex 스킬 디렉터리로 복사합니다.

- 본 스킬의 **docs/tools-repo-template/** 에 `setup.ps1`, `setup.sh`, `README.md` 예시가 있으므로, 도구 레포를 만들 때 그 폴더 내용을 복사해 사용할 수 있습니다.

**setup.ps1** 예시:

```powershell
# setup.ps1 - 도구 통합 레포에서 실행
$ErrorActionPreference = "Stop"
$RepoRoot = $PSScriptRoot
$CodexSkills = if ($env:CODEX_HOME) { Join-Path $env:CODEX_HOME "skills" } else { Join-Path $env:USERPROFILE ".codex\skills" }

if (-not (Test-Path $CodexSkills)) { New-Item -ItemType Directory -Path $CodexSkills -Force | Out-Null }

Get-ChildItem -Path (Join-Path $RepoRoot "skills") -Directory | ForEach-Object {
    $dest = Join-Path $CodexSkills $_.Name
    Write-Host "Deploying skill: $($_.Name) -> $dest"
    if (Test-Path $dest) { Remove-Item $dest -Recurse -Force }
    Copy-Item -Path $_.FullName -Destination $dest -Recurse -Force
}
Write-Host "Done. Restart Codex to load skills."
```

**setup.sh** (Mac/Linux):

```bash
#!/usr/bin/env bash
set -e
REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
CODEX_SKILLS="${CODEX_HOME:-$HOME/.codex}/skills"
mkdir -p "$CODEX_SKILLS"
for dir in "$REPO_ROOT/skills"/*/; do
  name=$(basename "$dir")
  echo "Deploying skill: $name -> $CODEX_SKILLS/$name"
  rm -rf "$CODEX_SKILLS/$name"
  cp -R "$dir" "$CODEX_SKILLS/$name"
done
echo "Done. Restart Codex to load skills."
```

- 필요하면 `cursor/rules`를 `~/.cursor/rules` 등으로 복사하는 줄을 같은 스크립트에 추가하면 됨.

### 2.4 README.md 작성

레포 루트의 **README.md**에 다음을 적어 두면, 다른 PC에서 clone 후 바로 따라 할 수 있습니다.

```markdown
# My Codex / Cursor Tools

스킬·플러그인 등 도구를 한 레포에 모아 두고, clone 후 setup으로 동일 환경을 구성합니다.

## 포함된 스킬

- **patent-strategy-report**: RFP 기반 특허 전략 보고서 (Google Patents CSV → 집계·보고서)

## 새 PC에서 환경 구성

1. **저장소 클론**
   ```bash
   git clone https://github.com/<사용자명>/my-tools.git
   cd my-tools
   ```

2. **도구 배포**
   - Windows: `.\setup.ps1`
   - Mac/Linux: `chmod +x setup.sh && ./setup.sh`
   - 이렇게 하면 `skills/` 안의 스킬들이 Codex 스킬 디렉터리(`~/.codex/skills` 또는 `%CODEX_HOME%\skills`)로 복사됩니다.

3. **Codex(또는 Cursor) 재시작**  
   새 스킬을 인식시키기 위해 한 번 재시작합니다.

4. **스킬별 의존성** (해당 스킬을 쓸 때만)
   - patent-strategy-report: `skills/patent-strategy-report/scripts/`에서  
     `uv venv && uv pip install -r requirements.txt` (또는 pip 동일)
   - 다른 스킬도 각자 `scripts/requirements.txt` 등이 있으면 동일하게 설치.

## 레포 구조

- `skills/` — Codex 스킬 (각 하위 폴더 = 스킬 하나)
- `cursor/` — (선택) Cursor 규칙 등
- `setup.ps1` / `setup.sh` — clone 후 한 번 실행해 도구 배포
```

### 2.5 Git 초기화 및 GitHub 푸시

```powershell
cd my-tools
git init
git add .
git commit -m "Initial: tools repo with patent-strategy-report skill"
git remote add origin https://github.com/<사용자명>/my-tools.git
git branch -M main
git push -u origin main
```

- GitHub에서 `my-tools` 저장소를 미리 만들어 두고, README/ .gitignore 없이 빈 저장소로 생성한 뒤 푸시하면 됨.

## 3. 다른 컴퓨터에서 사용하기

| 단계 | 작업 |
|------|------|
| 1 | `git clone https://github.com/<사용자명>/my-tools.git` 후 `cd my-tools` |
| 2 | Windows: `.\setup.ps1` / Mac·Linux: `./setup.sh` 실행 |
| 3 | Codex(또는 Cursor) 재시작 |
| 4 | 스킬을 실제로 쓰는 PC에서는, 해당 스킬의 `scripts/`에서 `uv pip install -r requirements.txt` 등으로 의존성 설치 |

이후에는 **레포만 clone + setup** 하면 동일한 스킬·도구 세트를 쓸 수 있습니다. 스킬을 추가·수정할 때는 이 레포를 수정하고 push한 뒤, 다른 PC에서는 `git pull` 후 필요 시 `setup`을 다시 실행하면 됩니다.

---

## 4. 요약

- **한 레포에 스킬·플러그인을 모으기**: `skills/`, `cursor/` 같은 하위 폴더에 각각 넣고, 루트에 `setup.ps1`·`setup.sh`를 둔다.
- **다양한 컴퓨터에 동일 환경 구성**: `git clone` → `setup` 실행 → (필요 시 스킬별 의존성 설치) → Codex/Cursor 재시작.
- **업데이트**: 레포에서 수정 후 push, 다른 PC에서는 `git pull` 후 필요 시 `setup` 재실행.

이렇게 정리하면 "도구 통합 레포지토리" 하나로 여러 PC의 도구 환경을 동일하게 유지할 수 있습니다.

---

## 5. 템플릿 파일 위치

이 스킬 안에 **docs/tools-repo-template/** 폴더가 있습니다. 여기에는 다음이 들어 있습니다.

- **setup.ps1** — Windows용 배포 스크립트
- **setup.sh** — Mac/Linux용 배포 스크립트  
- **README.md** — 도구 레포 루트용 README 예시

도구 통합 레포(`my-tools`)를 새로 만들 때, 위 파일들을 레포 루트로 복사한 뒤 `skills/`만 채우면 됩니다.
