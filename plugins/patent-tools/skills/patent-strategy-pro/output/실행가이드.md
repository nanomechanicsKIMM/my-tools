# Patent Strategy Pro – 단계별 실행 가이드

> **버전**: v2.0 (Agent/Subagent 아키텍처)
> Phase 4가 병렬 실행되어 3~5배 빠릅니다.

## 전체 워크플로 개요

```
[Orchestrator] SKILL.md
  │
  ├─ [Step 0] DB 선택 + 입력 정보 수집 (사용자 입력)
  │
  ├─ [Agent] Phase 1: phase1-rfp-prep       → output/rfp.md
  │
  ├─ [Agent] Phase 2A: phase2a-sub-tech-extract
  │                                          → output/sub_techs.json
  │                                          → 승인 테이블 반환
  │
  ├─ ⛔ USER APPROVAL GATE (사용자 확인 필수)
  │
  ├─ [Agent] Phase 2B: phase2b-query-gen
  │                                          → output/queries_sub_techs.md
  │                                          → output/acquisition_manifest.json
  │                                          → CSV 파일들
  │
  ├─ [PARALLEL — 단일 메시지로 동시 실행]
  │   ├─ [Agent] Phase 3: phase3-main-stats  → output/aggregate_report_data.json
  │   ├─ [Agent] Phase 4: sub1 분석         → output/sub1/core5_patents.csv
  │   ├─ [Agent] Phase 4: sub2 분석         → output/sub2/core5_patents.csv
  │   └─ [Agent] Phase 4: subN 분석         → output/subN/core5_patents.csv
  │
  └─ [Agent] Phase 5: phase5-report-writer  → output/{DATE}_{topic}_특허전략보고서.md
```

## 시작 방법

Claude에게 다음과 같이 요청하세요:

```
특허 전략 보고서를 만들어주세요.
RFP: C:\path\to\my_rfp.pdf
```

또는:

```
/patent-strategy-pro
```

스킬이 자동으로:
1. 특허 DB 선택 (Google Patents / EPO OPS) 질문
2. 필요한 입력 정보 수집
3. Phase 1→2A→(승인)→2B→(3+4 병렬)→5 순서로 에이전트 실행

## EPO OPS API 설정 (최초 1회, 자동 모드 사용 시)

```bash
# 1. https://developers.epo.org 에서 계정 등록 → 앱 생성 → Consumer Key/Secret 발급

# 2. 환경 변수 설정 (Windows)
set EPO_OPS_KEY=<consumer_key>
set EPO_OPS_SECRET=<consumer_secret>

# 영구 설정 (PowerShell)
[System.Environment]::SetEnvironmentVariable("EPO_OPS_KEY","<key>","User")
[System.Environment]::SetEnvironmentVariable("EPO_OPS_SECRET","<secret>","User")

# 3. 패키지 설치
pip install -r C:\Users\JHKIM\.claude\skills\patent-strategy-pro\scripts\requirements.txt
```

> **EPO OPS 무료 티어**: 쿼리당 최대 2,000건, 주간 4GB 다운로드.

## 핵심 설계 원칙

### acquisition_manifest.json — 단계 간 핸드오프 계약

Phase 2B 에이전트가 작성. Phase 3과 모든 Phase 4 에이전트가 이 파일만 읽어 CSV 경로를 파악.

```json
{
  "rfp_md": "output/rfp.md",
  "topic": "센서융합디스플레이",
  "date": "20260315",
  "db_mode": "google",
  "main_csv": "output/gp-search-20260315_main.csv",
  "sub_tech_csvs": {
    "sub1": "output/gp-search-20260315_sub1.csv",
    "sub2": "output/gp-search-20260315_sub2.csv",
    "sub3": "output/gp-search-20260315_sub3.csv"
  }
}
```

### Phase 3 + Phase 4 병렬 실행

오케스트레이터가 단일 메시지로 N+1개 에이전트를 동시 실행:
- Phase 3 (메인 통계): ~60초
- Phase 4 × N (세부 기술별): ~180초 각각, but 병렬이므로 총 ~180초

직렬 대비 3~5배 속도 향상.

## 트러블슈팅

| 문제 | 해결 |
|------|------|
| Phase 1 실패: PDF 변환 오류 | `pip install pdfplumber` 실행 후 재시도 |
| Phase 2A: 세부 기술 품질 낮음 | 승인 게이트에서 "수정: sub1 key_terms를 X, Y, Z로 바꿔주세요" 입력 |
| Phase 2B 실패: EPO 인증 오류 | `EPO_OPS_KEY`, `EPO_OPS_SECRET` 환경변수 확인 |
| Phase 2B (Google): CSV 경로 오류 | 파일명에 공백 없는지 확인, 절대 경로로 제공 |
| Phase 4 실패: CSV 없음 | acquisition_manifest.json의 sub_tech_csvs 경로 확인 |
| Phase 4 EPO 초록 403 오류 | EP/WO 외 특허는 fulltext 미제공, 빈 값으로 계속 진행됨 |
| scikit-learn 없음 | `pip install scikit-learn` |
| aggregate_csv_report import 오류 | `patent-strategy-report/scripts/` 경로 확인 |
