# 특허 전략 보고서 워크플로 요약

RFP 마크다운과 사용자 제공 CSV를 기준으로 **두 가지 데이터 경로** 중 하나를 선택해 집계·보고서까지 진행할 수 있다. 도메인에 관계없이 동일한 절차를 적용한다.

## 전체 경로 비교

| 구분 | 경로 A: 노이즈 필터 루프 | 경로 B: 연관성 상위 N건 |
|------|---------------------------|---------------------------|
| **입력** | v1 CSV (사용자 다운로드) | v1 CSV (사용자 다운로드) |
| **1단계** | `filter_noise.py` → 제거율 계산 | `filter_title_top_n.py` → 제목–RFP 연관성 상위 5000건 (선택: `--exclude-terms` 로 제목 제외 키워드 적용) |
| **반복** | REMOVAL_RATIO > 10% 시 NOT 보정 → 재검색·재다운로드·재실행 | 없음 (상위 N건으로 고정) |
| **선택** | — | `fetch_abstracts.py` → `filter_abstract_top_n.py` → 초록 기준 상위 2500건 (v2). 핵심 특허(예: 상위 10건) 선정 시 제목·초록 **제외 키워드** 필터 적용 후 상위 N건 저장 |
| **공통** | 집계(`aggregate_csv_report.py`) → 보고서 채우기(`fill_report.py`) → 서술문 LLM 작성. **국가별**은 유럽 통합·한글 명칭 사용(`aggregate_countries_for_report`) |

## 공통 전제

1. **검색식·URL**: `generate_query.py <RFP.md> [기술분류] [--years N]`  
   - 현재 폴더 RFP: 첫 인자 `"."` → `*RFP*.md` 자동 탐색.  
   - 핵심 키워드: `--required-terms "a,b,c"`, `--required-all`, `--ask-required`.  
   - **제외 키워드**: `--exclude-terms "term1,term2"` (예: 디스플레이 RFP에서 `"OLED,LCD"`) → 검색식에 NOT 추가; 연관성 필터·핵심 특허 선정에서도 동일 키워드로 제목/초록 필터 적용 권장.
2. **CSV**: 사용자가 Google Patents에서 검색 URL로 검색 후 **Download (CSV)** 로 저장한 파일 경로 제공.
3. **집계**: 출원인 **영어 통일·동일 회사 합산** 적용 (법인 접미사 처리, CANONICAL_APPLICANT_EN). **국가별** 보고서용은 **유럽 통합·한글 명칭**(`aggregate_countries_for_report`) 사용.
4. **보고서**: 기술 단계(도입기/성장기/성숙기)는 RFP·도메인에 맞게 해석; 최근년도 감소 시 공개 지연·표본 한계 구분.

## 스크립트 역할 요약

| 용도 | 스크립트 |
|------|----------|
| 검색식·URL | `generate_query.py` |
| 경로 A 노이즈 필터 | `filter_noise.py` |
| 경로 B 제목 상위 N | `filter_title_top_n.py` |
| 경로 B 초록 추가 | `fetch_abstracts.py` |
| 경로 B 초록 상위 N | `filter_abstract_top_n.py` |
| 경로 B 일괄 | `run_v1_to_v2_pipeline.py` |
| 집계 | `aggregate_csv_report.py` |
| 보고서 채우기 | `fill_report.py` |
| 경로 A 전체 파이프라인 | `run_full_pipeline.py` |
| 5000건 기준 집계·보고서(경로 내부 경로 지정) | `run_aggregate_5000_inprocess.py`, `run_report_from_5000.py` |
