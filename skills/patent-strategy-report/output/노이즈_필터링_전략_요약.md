# 노이즈 필터링 전략 요약

현재 스킬에서 **노이즈**를 다루는 방식은 **두 가지 경로**로 나뉩니다. 아래는 그 핵심만 요약한 것이며, **수정이 필요하면 의견을 주시면 반영하겠습니다.**

---

## 1. 노이즈의 정의 (공통)

- **노이즈**: RFP의 기술 목표와 **관련성이 낮은 특허**
  - 다른 적용/기술 분야이거나, RFP 핵심 키워드와 제목·초록 겹침이 적어 “연관성 점수”가 낮게 나오는 행
- 즉, “검색에는 걸렸지만 RFP 관점에서는 제외하고 싶은 특허”를 노이즈로 본다.

---

## 2. 경로 A: 노이즈 필터 루프 (반복 보정)

**목표**: 제거 비율(REMOVAL_RATIO)이 **10% 이하**가 될 때까지 검색식을 NOT으로 보정하며 반복.

### 2.1 한 라운드에서 하는 일

| 항목 | 내용 |
|------|------|
| **판별 방식** | **키워드 겹침 비율**: RFP에서 추출한 키워드(영문·한글·RFP명 등)가 각 특허의 제목+(있으면)초록에 얼마나 나오는지 비율로 점수화 |
| **점수 식** | `keyword_score = (텍스트에 등장한 키워드 수) / (전체 RFP 키워드 수)` |
| **기준값** | `--threshold` (기본 **0.15**). 점수 **≥ threshold** → 유지(keep), **< threshold** → 노이즈(제거) |
| **출력** | 유지된 행만 저장한 CSV, **REMOVAL_RATIO** = 제거 건수 / 전체 건수 |

### 2.2 반복 조건 및 NOT 제안

- **REMOVAL_RATIO > 10%** 이면:
  - **제거된 행(노이즈)** 의 제목·초록에서 단어 빈도 산출
  - **유지(keep) 행**에서는 상대적으로 적게 나온 단어만 NOT 후보로 선택
  - RFP 키워드·현재 검색식에 있는 단어·stopword(method, device, system, sensor 등)는 제외
  - 상위 N개(기본 5개)를 **SUGGESTED_NOT_TERMS** 로 제안
  - **IMPROVED_QUERY_SUGGESTION** = `현재 검색식 + " NOT (term1 OR term2 OR ...)"` 출력
- 사용자: 제안된 검색식/URL로 **재검색 → 새 CSV 다운로드 → 동일 필터 재실행**. **REMOVAL_RATIO ≤ 10%** 될 때까지 반복.

### 2.3 대안: 초록 기반 NOT 품질 향상

- CSV에 초록이 없을 때: **초록 크롤링** → **초록–RFP TF-IDF 점수**로 노이즈 분류(`score_relevance.py`) → **노이즈 행에서 NOT 후보 추출**(`extract_not_from_noise.py`).
- 한 번에 실행: `run_noise_pipeline_with_abstracts.py` (초록 수집 후 점수·NOT 추출).

### 2.4 경로 A 요약 파라미터

| 파라미터 | 기본값 | 의미 |
|----------|--------|------|
| threshold | 0.15 | 키워드 겹침 비율; 이 이상이면 유지 |
| REMOVAL_RATIO 목표 | ≤ 10% | 이하면 루프 종료, 집계·보고서로 진행 |
| suggest-top-n | 5 | NOT 후보 최대 개수 |

---

## 3. 경로 B: 연관성 상위 N건 + 제외 키워드 (반복 없음)

**목표**: “노이즈 루프” 없이, **연관성 점수 상위 N건**만 남기고, RFP 범위에서 **제외하고 싶은 키워드**는 검색식·필터·핵심 특허 선정에서 일관되게 제외.

### 3.1 제목 단계

- **filter_title_top_n.py**: 제목–RFP **TF-IDF 코사인 유사도**로 점수 부여 → **상위 N건**만 저장.
- **(선택) 제외 키워드**: `--exclude-terms "term1,term2"` (예: OLED, LCD)
  - **제목**에 해당 단어가 포함된 행은 **점수 계산 전에 제거** 후, 남은 행만으로 상위 N건 산출.
  - 검색식에 NOT을 넣었어도 수집 CSV에 섞여 들어온 경우를 보정.

### 3.2 초록 단계 (선택)

- **fetch_abstracts.py**로 초록 추가 → **filter_abstract_top_n.py**로 초록–RFP TF-IDF 점수 재계산 → 점수순 상위 M건(예: 2500건) 저장.

### 3.3 핵심 특허 선정 (예: 상위 10건)

- 점수순 정렬된 CSV에서 **제목 또는 초록**에 제외 키워드(예: OLED, LCD)가 포함된 행을 **제거한 뒤** 상위 N건을 핵심 특허로 저장.
- 검색식·제목 필터·핵심 특허에서 **동일 제외 키워드**를 쓰면 정합성 유지.

### 3.4 경로 B 요약

- **반복 없음**: 한 번에 상위 N건으로 좁힘.
- **노이즈 정의**: “연관성 점수가 낮은 것”이 아니라 “상위 N건 밖” + “제외 키워드 포함 행”.
- **제외 키워드**: 검색식(`generate_query.py --exclude-terms`) → 제목 필터(`--exclude-terms`) → 핵심 특허 선정(동일 목록)으로 **3단계 일치** 권장.

---

## 4. 두 경로 비교

| 구분 | 경로 A (노이즈 필터 루프) | 경로 B (연관성 상위 N건) |
|------|---------------------------|---------------------------|
| **노이즈 판별** | RFP 키워드 겹침 비율 < threshold(0.15) | 상위 N건 밖 + (선택) 제목/초록 제외 키워드 |
| **반복** | REMOVAL_RATIO > 10% 시 NOT 제안 → 재검색·재다운로드·재실행 | 없음 |
| **종료 조건** | REMOVAL_RATIO ≤ 10% | 상위 N건 추출 완료 시 |
| **NOT/제외** | 노이즈 행에서 자동 추천 NOT 후보 | 사용자/도메인별로 제외 키워드 지정 (--exclude-terms) |

---

## 5. 현재 스킬에서의 위치

- **경로 A**: `filter_noise.py`, `run_full_pipeline.py`, `score_relevance.py`, `extract_not_from_noise.py`, `run_noise_pipeline_with_abstracts.py`
- **경로 B**: `filter_title_top_n.py` (--exclude-terms), `filter_abstract_top_n.py`, 파이프라인 내 “핵심 특허 선정 시 제목·초록 제외” 로직

---

## 6. 의견 요청

다음에 대해 **수정·보완하고 싶은 점**이 있으면 알려주세요.

1. **경로 A**
   - threshold(0.15), REMOVAL_RATIO 10% 기준을 바꾸고 싶은지 (예: 더 엄격/완화).
   - 노이즈 판별을 “키워드 겹침” 대신 **TF-IDF 코사인** 등으로 통일하고 싶은지.
   - NOT 후보 개수(suggest-top-n), stopword 목록 추가/삭제.

2. **경로 B**
   - “상위 N건”만으로 두고 제외 키워드는 선택으로 유지할지, 아니면 경로 B에도 “노이즈 비율” 개념을 도입할지.
   - 제외 키워드를 RFP/설정 파일에서 읽어와 자동 적용하는 방식이 필요한지.

3. **공통**
   - “노이즈”라는 용어를 경로 A에만 쓰고, 경로 B는 “연관성 필터·제외 키워드”로만 부를지.
   - 두 경로를 하나의 “통합 노이즈 전략”으로 정리할지(예: 1차 연관성 상위 N → 2차 키워드 노이즈 제거 → REMOVAL_RATIO 확인 등).

의견 주시면 그에 맞춰 스킬·reference·스크립트 반영 방안을 제안하겠습니다.
