## 3. 다음 작업 단계 (권장 순서)

| 순서 | 단계 | 담당 | 내용 |
|------|------|------|------|
| **1** | 검색식·URL 생성 | 사용자/에이전트 | `generate_query.py <RFP.md> [기술분류] [--years N]` → SEARCH_URL 복사 |
| **2** | CSV 1차 다운로드 | 사용자 | 브라우저에서 SEARCH_URL 열기 → **Download (CSV)** 클릭 → 저장 경로 확보 |
| **3** | 파이프라인 1차 실행 | 사용자/에이전트 | `run_full_pipeline.py <CSV경로> --rfp-path <RFP.md> --search-query "..." --search-url "..."` |
| **4** | 노이즈 루프 (필요 시 반복) | 사용자/에이전트 | REMOVAL_RATIO > 10%이면 출력된 IMPROVED_SEARCH_URL로 재검색·CSV 재다운로드 후 3번 다시 실행. ≤10% 될 때까지 반복 |
| **5** | 보고서 서술문 채우기 | 사용자/LLM | 생성된 MD에서 `[ ... ]` 플레이스홀더를 LLM으로 채워 최종본 저장 |
| **6** | 품질 검토 (선택) | 사용자 | 샘플과 비교, 수치 검증, 노이즈 기준·reference.md 조정 |

### 3.1 즉시 할 일 (사용자)

1. **RFP 확정** 후 `generate_query.py`로 검색식·URL 생성.
2. **Google Patents**에서 해당 URL로 검색 → **Download (CSV)** → CSV 경로 확보.
3. **run_full_pipeline.py** 실행 시 `--search-query`와 `--search-url`을 반드시 넘겨, 노이즈 루프 시 개선 검색식·URL을 받을 수 있게 함.

### 3.2 파이프라인 실행 (CSV 제공 후)

```bash
cd .codex\skills\patent-strategy-report\scripts
uv run python run_full_pipeline.py "<CSV경로>" --rfp-path "<RFP.md>" --search-query "<검색식>" --search-url "<URL>" [--report-title "제목"] [--topic 주제]
```

- **REMOVAL_RATIO > 10%**이면: 출력된 **IMPROVED_SEARCH_URL**로 브라우저 재검색 → CSV 재다운로드 → 위 명령에서 CSV 경로와 `--search-query`/`--search-url`만 개선된 값으로 바꿔 다시 실행.
- **REMOVAL_RATIO ≤ 10%**이면: 집계·보고서까지 자동 완료.

### 3.3 서술문 채우기

생성된 보고서 MD에서 `[ ... ]` 안 개요·해석·시사점 등을 LLM으로 1~2문단씩 작성해 최종본 완성.

### 3.4 품질 검토 (선택)

샘플과 비교해 섹션·표·수치·노이즈 기준을 점검하고 reference.md·스킬을 수정.
