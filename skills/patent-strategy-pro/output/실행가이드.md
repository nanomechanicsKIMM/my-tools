# Patent Strategy Pro – 단계별 실행 가이드

## 전체 워크플로 개요

```
[Phase 1] RFP 준비         → PDF→MD 변환 (필요 시)
[Phase 2] 세부 기술 도출   → extract_sub_technologies.py + Claude 보정
[Phase 3] EPO OPS 자동 검색 → run_pipeline.py --auto-download (API 키 필요)
          └ 대안: Google Patents 수동 CSV 다운로드
[Phase 4] 메인 통계 분석   → 10,000건 기반 통계
[Phase 5] 세부 기술 분석   → 세부 기술별 핵심 특허 5건 (초록 EPO OPS 수집)
[Phase 6] 보고서 작성      → Claude가 §1~§9 작성
```

---

## EPO OPS API 설정 (최초 1회)

```bash
# 1. https://developers.epo.org 에서 계정 등록 → 앱 생성 → Consumer Key/Secret 발급

# 2. 환경 변수 설정 (Windows)
set EPO_OPS_KEY=<consumer_key>
set EPO_OPS_SECRET=<consumer_secret>

# 영구 설정 (PowerShell)
[System.Environment]::SetEnvironmentVariable("EPO_OPS_KEY","<key>","User")
[System.Environment]::SetEnvironmentVariable("EPO_OPS_SECRET","<secret>","User")

# 3. 패키지 설치
pip install python-epo-ops-client
```

> **EPO OPS 무료 티어**: 쿼리당 최대 2,000건, 주간 4GB 다운로드. 대부분의 RFP 분석에 충분.

## Phase 2: 검색식 생성

### 2-A. 세부 기술 도출 (자동 추출 + 반드시 검토) — D-1

```bash
python extract_sub_technologies.py "rfp.md" -o "output/sub_techs.json"
```

스크립트 실행 완료 후 **Claude가 자동으로 추출 결과를 아래 형식으로 제시합니다**:

```
## 세부 기술 자동 추출 결과 확인

| ID | 한국어 기술명 | Google Patents 검색 키워드 |
|----|--------------|--------------------------|
| sub1 | {name_ko} | {key_terms} |
| sub2 | {name_ko} | {key_terms} |
| ...  | ...         | ...                       |

검토 포인트:
- 기술명이 RFP 연구개발내용 항목과 일치하는가?
- 중요한 세부 기술이 빠져 있지 않은가?
- key_terms가 Google Patents 검색에 적합한 영어 기술 용어인가?
- 세부 기술별 key_terms가 서로 다른가? (모두 동일하면 검색 품질 저하)

위 목록이 맞으면 "확인" 또는 수정 내용을 알려주세요.
```

> **⛔ 중요**: Claude는 사용자의 명시적 승인("확인", "OK", "진행", "맞습니다") 전에는
> 절대로 Phase 2 (검색식 생성)로 진행하지 않습니다.

`sub_techs.json` 검토·수정 체크리스트:
- [ ] `name_ko`: 간결한 기술명인가? (전체 문장 X, 20자 이내 기술명 권장)
- [ ] `name_en`: 적절한 영문 기술명인가?
- [ ] `key_terms`: 3~8개의 Google Patents 검색 영어 키워드 (세부 기술별로 달라야 함)
- [ ] `exclude_terms`: 노이즈 방지를 위한 제외 단어
- [ ] `rfp_objectives`: 연관된 RFP 목표/성과지표 항목
- [ ] 세부 기술 개수가 RFP 연구개발내용과 일치하는가?

> **팁**: key_terms가 모든 세부 기술에서 동일하다면 반드시 수정하세요.
> 각 세부 기술별로 특화된 영어 기술 용어를 지정해야 검색 품질이 높아집니다.
> 예: TFT 백플레인 → "TFT", "flexible backplane", "LTPS", "OLED backplane"
>     센서 어레이 → "strain sensor", "sensor array", "deformation sensing", "piezoelectric"
>     UI/UX → "user interface", "gesture recognition", "deformation interaction", "haptic"

### 2-B. 검색식 생성

```bash
# 메인 검색식 + 세부 기술별 검색식 동시 생성
python generate_query.py "rfp.md" \
  --sub-tech-json "output/sub_techs.json" \
  --years 10 \
  -o "output/queries_sub_techs.md"
```

출력:
- `output/queries_sub_techs.md` – 전체 검색식 모음
- 터미널에 각 세부 기술별 검색 URL 출력

### 2-C. EPO OPS 자동 검색 (권장)

EPO OPS API 키가 설정된 경우 검색부터 초록 수집까지 전체 자동화:

```bash
# Phase 3+4+초록 수집 전체 자동 실행
python run_pipeline.py \
  --rfp "output/rfp.md" \
  --sub-tech-json "output/sub_techs.json" \
  --auto-download \
  --topic "센서융합디스플레이" \
  --core-n 5 \
  -o "output/"
```

- EPO OPS로 메인+세부기술 검색 → CSV 자동 생성
- EPO OPS로 초록+대표청구항 수집 (Google Patents 스크래핑 불필요)
- `--split-by-year` 옵션: 결과 >2,000건일 때 연도별 분할 검색

### 2-C (대안). Google Patents 수동 다운로드

EPO OPS 계정이 없는 경우 기존 방식 사용:

1. `queries_sub_techs.md`의 Google Patents URL 각 항목별로 브라우저에서 열기
2. 결과 목록 상단 **Download (CSV)** 클릭 → `main.csv`, `sub1.csv`, ... 저장
3. Phase 3, 4를 수동 CSV로 실행

> Google Patents 최대 1,000건. 초록 수집 시 `fetch_abstracts.py`가 Google Patents를 스크래핑하므로 rate limit 주의.

## Phase 3+4: 메인 통계 및 세부 기술 분석

### EPO OPS 자동 모드 (위 2-C 권장 명령어로 통합 실행)

```bash
# --auto-download가 Phase 3+4 모두 처리
python run_pipeline.py \
  --rfp "output/rfp.md" \
  --sub-tech-json "output/sub_techs.json" \
  --auto-download \
  --topic "기술명" \
  -o "output/"
```

### 수동 모드 (Google Patents CSV 사용)

```bash
# Phase 3: 메인 CSV 통계
python run_pipeline.py \
  --rfp "output/rfp.md" \
  --main-csv "main.csv" \
  --topic "센서융합디스플레이" \
  -o "output/"

# Phase 4: 세부 기술 분석
python run_pipeline.py \
  --rfp "output/rfp.md" \
  --sub-tech-json "output/sub_techs.json" \
  --sub-tech-csvs "sub1.csv,sub2.csv,sub3.csv" \
  --topic "센서융합디스플레이" \
  --core-n 5 \
  -o "output/"
```

> CSV 순서는 `sub_techs.json`의 `sub_technologies` 배열 순서와 일치해야 합니다.

출력 (세부 기술별):
```
output/
├── sub1/
│   ├── top100_title_scored.csv
│   ├── top100_with_abstracts.csv   (초록+대표청구항 포함)
│   ├── top100_abstract_scored.csv
│   └── core5_patents.csv           ← 핵심 특허 5건
├── sub2/
│   └── ...
├── sub3/
│   └── ...
└── sub_tech_core_summary.json
```

### Phase 4 재개 (초록 수집 중단 시)

```bash
# --skip-fetch: 이미 수집된 abstracts CSV 재사용
python run_pipeline.py \
  --rfp "output/rfp.md" \
  --sub-tech-json "output/sub_techs.json" \
  --sub-tech-csvs "sub1.csv,sub2.csv,sub3.csv" \
  --skip-fetch \
  -o "output/"
```

Phase 3~4 완료 후 Claude에게 요청:

Claude가 순서대로 작성:

출력: `output/{YYYYMMDD}_{topic}_특허전략보고서.md`

## 트러블슈팅

| 문제 | 해결 |
|------|------|
| EPO OPS 인증 실패 | `EPO_OPS_KEY`, `EPO_OPS_SECRET` 환경변수 설정 확인. `developers.epo.org` 계정 활성화 여부 확인 |
| EPO OPS 결과 0건 | CQL 검색식에 `ti=` 필드 접두사 누락 여부 확인. `--split-by-year` 옵션 시도 |
| EPO OPS 2000건 한계 | `--split-by-year` 로 연도별 분할 검색 (자동으로 연도별 재검색 후 합산) |
| EPO 초록 수집 실패 | EP/WO 특허 외에는 fulltext 미제공 경우 있음. 빈 값으로 저장 후 계속 진행 |
| PDF 변환 시 텍스트 깨짐 | `pip install pdfplumber` 버전 업. 또는 직접 복사+붙여넣기 후 MD 작성 |
| 세부 기술 도출 품질 낮음 | `sub_techs.json` 직접 편집 or Claude에게 RFP 읽고 재작성 요청 |
| `scikit-learn` 없음 | `pip install scikit-learn` |
| `aggregate_csv_report` import 오류 | `patent-strategy-report/scripts/` 경로가 정확한지 확인 |
