# 특허 전략 보고서 자동 생성 스킬

R&D 과제 기획자가 Google Patents에서 CSV를 다운로드하면, 연도별 추이·주요 출원인·국가별 분포를 분석하여 한국어 Obsidian 보고서를 생성한다.

## 워크플로우 개요

```
PHASE 1   → 초기 검색식 생성 + Google Patents URL 제공 + CSV 다운로드
              ↕  (노이즈 수준이 허용 가능할 때까지 반복)
PHASE 1.5 → 샘플 타이틀 진단 → 검색식 NOT절 추가 → URL 재생성 → 재다운로드
PHASE 2   → 최종 CSV 분석 → 보고서 초안 생성 → 전략 시사점 작성 → 파일 저장
```

노이즈는 다운로드 후 후처리로 제거하지 않는다. **검색식 자체를 정밀화**하여 소스에서 차단하는 것이 원칙이다. 이렇게 해야 집계 통계의 신뢰도가 유지되고, 검색식이 재현 가능한 자산이 된다.

---

## PHASE 1: 초기 검색식 생성

사용자가 RFP 또는 기술 키워드를 제공하면 Google Patents Boolean 검색식을 생성한다.

### 검색식 설계 원칙

좋은 검색식은 관련 특허를 빠짐없이 잡으면서(recall), 무관한 도메인 노이즈를 줄이는(precision) 균형을 맞춘다.

1. **기술 핵심어**: RFP의 기술 구조 명사를 영문으로 추출 (OR로 연결)
2. **응용 맥락**: 목표 응용 분야 키워드 (AND로 한정)
3. **제외 키워드**: NOT으로 예측 가능한 무관 도메인 제거

### Google Patents URL 형식

```
https://patents.google.com/?q=<URL인코딩된 검색식>&before=priority:<종료연도>1231&after=priority:<시작연도>0101
```

### CSV 다운로드 안내

```
📥 CSV 다운로드 방법:
1. 위 URL에서 검색 결과 확인
2. 우측 상단 [↓ Download] 버튼 클릭 → "CSV" 선택
3. CSV 파일 경로를 알려주시면 노이즈 진단을 시작합니다.
```

## PHASE 1.5: 검색식 정밀도 개선 (반복 루프)

CSV를 받으면 먼저 샘플 타이틀을 진단해 검색식의 false positive 원인을 파악한다. 이 과정을 노이즈가 허용 가능한 수준이 될 때까지 반복한다.

### Step 1: 샘플 진단

```bash
PYTHONUTF8=1 uv run python ~/.claude/skills/patent-report/scripts/csv_analyzer.py \
  --csv-path "<CSV 경로>" \
  --show-sample 50
```

### Step 2: 노이즈 원인 진단

샘플의 각 false positive 타이틀에 대해 반드시 진단한다:

> **"이 특허가 검색식에 걸린 이유는 무엇인가?"**
> → 현재 검색식의 어떤 키워드가 이 false positive를 유발했는가?

이 진단이 핵심이다. 원인을 알아야 올바른 수정이 가능하다.

예를 들어:
- 키워드가 의도한 분야 외에서도 흔히 쓰이는 일반 명사라면 (예: `cell`이 배터리 셀과 생물학 세포 양쪽에 해당하는 경우) → 더 구체적인 복합어로 교체하거나, 해당 무관 도메인을 NOT으로 차단
- 응용 맥락 키워드가 너무 넓어 목표와 무관한 산업군까지 포함한다면 → AND 조건을 추가해 범위를 좁히거나, 무관 산업군을 NOT으로 제외

### Step 3: 체계적 노이즈 판별

샘플에서 **같은 원인**의 false positive가 3건 이상 발견되면 체계적 노이즈로 판단하고 검색식 수정 대상으로 지정한다. 1~2건의 산발적 노이즈는 무시한다.

### Step 4: 검색식 NOT절 추가 및 URL 재생성

체계적 노이즈마다 NOT절을 추가한 개선 검색식을 생성하고, 새 Google Patents URL을 제공한다.

```
개선 검색식 제시 형식:
- 수정 전: (기존 검색식)
- 수정 후: (개선 검색식)
- 수정 근거: 어떤 노이즈를 어떤 NOT절로 차단했는지 설명
```

사용자가 새 CSV를 다운로드한 후, Step 1~4를 반복한다.

### PHASE 2 진입 기준

샘플 50건에서 명확한 노이즈 비율이 약 10% 이하이거나, 사용자가 "이 정도면 충분하다"고 판단하면 PHASE 2로 진행한다.

## PHASE 2: CSV 분석 및 보고서 생성

### Step 1: 분석 실행

```bash
PYTHONUTF8=1 uv run python ~/.claude/skills/patent-report/scripts/csv_analyzer.py \
  --csv-path "<CSV 경로>" \
  --technology "<기술명 (한국어)>" \
  --search-query "<사용한 검색식>" \
  --analysis-period "<시작연도>-<종료연도>" \
  --output "<출력 파일 경로.md>" \
  [--exclude-keywords "키워드1,키워드2"] \
  [--include-keywords "키워드1,키워드2"] \
  [--tags "태그1,태그2"]
```

### Step 2: 초안 읽기 및 `_fill_` 항목 작성

스크립트가 `_fill_` 플레이스홀더를 남겨두는 전략적 분석 섹션을 Claude가 직접 작성한다.

**작성 대상 섹션**:

**§3.3 주요 출원인 전략 특성** — 상위 출원인을 국가 그룹으로 묶어 각 그룹의 기술 전략 패턴을 서술한다.

**국가 그룹 분류 기준**: 특허 ID 앞 2자리(출원 국가)가 아니라 출원인 이름으로 기업 국적을 판단한다.
- **한국**: 삼성, LG, 현대, 한글 표기 기업명 (예: 삼성디스플레이 주식회사 = Samsung Display Co.)
- **중국**: BOE(京东方), Huawei(华为), TCL, CSOT(华星光电), 한자 표기 기업명
- **미국**: Apple, Google, Microsoft, Corning, 3M 등
- **일본**: Sony, Panasonic, Sharp, Nippon 등 + 일본 한자 표기 기업명

**다국어 표기 통합**: csv_analyzer.py의 정규화는 법인 접미어(CO LTD/주식회사 등)는 제거하지만 한/영 혼재는 통합하지 못한다. 보고서에서 직접 통합 순위를 계산해 제시한다.
예: "삼성디스플레이 주식회사 XXX건 + Samsung Display Co. YYY건 = 실질 ZZZ건(1위)"

**각 그룹별 서술 프레임**:
- 그룹 내 상위 기업의 출원 비중 (전체 대비 %)
- 기술 집중 영역 패턴 (타이틀 샘플 기반 추론)
- 자국 중심 출원 vs 다국 분산 출원 전략 여부

**§4.2 국가별 전략 분석** — 상위 국가별로 출원 전략의 특성을 분석한다 (예: 미국=시장 방어, 중국=양적 추격, 일본=소재·원천, 한국=원천+다국 출원). KR 코드 출원 비중과 실질 한국 기업의 영향력(다국 분산 출원 고려) 간의 차이도 짚는다.

**§5. 종합 전략적 시사점** — 3가지 관점으로 작성:
1. **기술 발전 단계 판단**: 공개일 추이 데이터를 근거로 태동기/성장기/성숙기 중 어디에 해당하는지
2. **지정학적 IP 경쟁 구도**: 주요 국가·기업 간 관계 (협력/경쟁/틈새)
3. **R&D 과제 기획 시사점**: 아래 RFP 연계 지침 참조

시사점 작성 시 데이터에서 읽히는 실제 패턴에 기반해야 한다 — 일반적인 조언이 아니라, 이 기술 분야의 이 데이터가 말하는 것을 써야 한다.

---

### §5.3 R&D 과제 기획 시사점 — RFP 연계 작성 원칙

§5.3은 특허 데이터를 R&D 기획 맥락에 연결하는 핵심 섹션이다. RFP(과제기획요구서) 제공 여부에 따라 접근 방식이 달라진다.

#### RFP가 제공된 경우

RFP 문서를 읽고 다음 4가지 질문에 답하며 시사점을 도출한다. 이 질문들은 기술 도메인과 무관하게 어떤 RFP에나 적용된다.

**질문 1 — 이 RFP가 목표로 하는 기술 조합은 특허 데이터에서 얼마나 포화되어 있는가?**

특허 검색식은 대개 넓은 기술 영역을 커버하지만, RFP는 그 안의 특정 기술 조합(예: A + B + C의 통합 구현)을 목표로 한다. 해당 조합이 현재 특허 포트폴리오에서 충분히 표현되어 있는지 판단한다. 포화되지 않은 조합이 있다면 그것이 선점 기회다.

**질문 2 — RFP의 정량 성과목표가 현재 특허 선행기술과 어떤 관계에 있는가?**

RFP는 보통 성능 수치(예: 효율, 용량, 정밀도, 처리속도, 내구성, 감도 등 기술 특성에 맞는 지표)를 명시한다. 이 수치가 현재 알려진 선행 기술 대비 어느 수준인지 가늠한다. 세계 최고 수준을 초과하는 목표치는 달성 시 그 수치 범위 자체가 강력한 특허 청구항의 한정 요소가 되므로, 연구 초기부터 특허 명세서 작성을 염두에 두고 실험 데이터를 기록해야 한다.

**질문 3 — RFP가 요구하는 특허 출원 조건은 무엇이며, 특허 데이터의 국가별 전략과 어떻게 맞출 것인가?**

많은 RFP는 삼극특허(US+EP+JP), PCT 출원, 또는 출원 건수 목표를 명시한다. 특허 데이터에서 확인된 선도 기업들의 다국 출원 경로(§4.2)를 참고하여, 이 과제에 최적화된 출원 루트와 일정을 제안한다.

**질문 4 — RFP가 명시한 응용 분야에서 특허 공백은 어디에 있는가?**

RFP의 기대효과나 목표 시장(예: 웨어러블, 의료기기, 자동차, 에너지 등)을 특허 데이터의 출원인·국가 분포와 교차 분석한다. 주요 기업들이 아직 그 응용 도메인에 집중 출원하지 않은 영역이 있다면, 연구 결과를 해당 응용 맥락으로 특허화하는 전략을 권고한다.

이 4가지 질문에 기반해 **구체적인 행동 권고 4~6가지**를 작성한다.

**작성 형식**: 각 권고는 아래 구조를 따른다.
> **[권고 제목]**: [특허 데이터의 구체적 수치나 패턴] → [과제에서 취할 행동]

예시:
> **PCT 출원 우선 확보**: 한국(KR) 45% · 미국(US) 28% · 중국(CN) 18% 순으로 시장이 형성됨. 과제 착수 후 18개월 내 PCT 출원을 통해 US/CN 동시 진입 경로를 확보할 것.

> **[회사명] 특허 회피 설계**: 상위 출원인 [회사명]이 전체 출원의 XX%를 차지하며 [기술영역]에 집중 출원 중. 해당 영역 청구항 범위를 분석하여 회피 설계 검토 필요.

**RFP 수치 성과목표 인용 방법**: RFP의 정량 목표가 있으면 해당 수치를 보고서에 직접 인용한다.
예: "본 RFP는 게이지 팩터 500 이상을 목표치로 명시한다. 검색된 선행 특허 기준 현재 기술 수준은 ~300 수준이며, 500 이상 범위를 최초 청구항으로 선점할 가능성이 있다."

#### RFP가 제공되지 않은 경우

다음 4가지를 작성한다:
- **공백 영역**: 특허 데이터에서 상대적으로 출원이 적은 기술·응용 조합
- **선점 기회**: 출원 트렌드 변화(우선일 피크 이후 감소 구간 등)에서 읽히는 타이밍
- **다국 출원 전략**: 선도 기업들의 출원 패턴을 참고한 국가별 출원 권고
- **모니터링 대상 기업**: 특허 급증 중인 후발 주자 또는 주목해야 할 경쟁 출원인

### Step 3: 최종 파일 저장


## 주요 제약 사항 (사용자에게 안내)

- **CSV 다운로드**: Google Patents의 Download → CSV로 전체 검색 결과를 다운로드할 수 있음. 복수 파일 병합 시 쉼표로 경로 구분
- **공개 지연**: 최근 2년(우선일 기준) 출원 건수는 미공개로 인해 과소 표시됨 → 보고서 §2.3에 자동 명시
- **출원인 정규화**: 같은 기업의 다양한 표기명이 분산될 수 있음 → 보고서에서 직접 통합 언급
